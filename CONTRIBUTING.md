# Contributing to Astravue MCP Server

Thank you for your interest in contributing to the Astravue MCP Server.

## How to contribute

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Submit a pull request

## What we accept

- Bug fixes and documentation improvements
- New skills that automate common project management workflows
- Improvements to existing skill instructions

## What we don't accept

- Changes to the MCP server endpoint or authentication
- Skills unrelated to project management

## Code of conduct

Please read [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) before contributing.

## Questions?

Open an issue or email support@astravue.com.
